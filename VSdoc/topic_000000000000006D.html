Gadget Class

See Also

Base class for a Gadget. All Gadgets are one of insoluble, unsolved, or solved. This means a Gadget has three possible states, but it can only transition along edges in the I-U-S graph. How a Gadget reacts to being operated depends on its state. You control the reaction by providing definitions for each of the four virtual methods that match each state, in your subclass. A Gadget is initially unsolved and unlocked. If it has depedencies, it will be locked until all Gadgets it depends on are solved, at which point it becomes unlocked. Note that a Gadget actually can be solved in its initial state, but it is unsolved before it is enabled. This allows Gadgets to be enabled in any order, so an unenabled Gadget will be unsolved when a dependent attaches to it. When that Gadget is later enabled, it will send a SolvedChanged event to its dependents. If it is already enabled, it will be solved when a dependent enabled later attaches to it.
