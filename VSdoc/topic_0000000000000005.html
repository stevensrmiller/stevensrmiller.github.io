ConditionList Class

See Also

Maintains a list of Gadgets that all report when their status changes from completed to not completed, or vice versa. Also maintains a list of actions. The actions are invoked when the completion requirements of the list are met, which can be when all Gadgets report that they are completed, or when any one of them reports that it is completed, base on the setting of the allMustBeCompleted flag.

Fields
Name Description
actionsOnConditionsCompleted Any public method can be called from this action list, by adding it to the list in the Unity Inspector. Note that the state of the entire list is reported. That is, if the list requires that all Gadgets be completed (that is, when allMustBeCompleted is true), then when all Gadgets are completed, the actions are all called and are passed the boolean value true. In that case, if one Gadget later changes from being completed to no longer being completed, all of the actions are called again, this time being pass the boolean value false. Similarly, if allMustBeCompleted is false, then when all Gadgets are not completed, and then one becomes completed, all actions are called and are passed the boolean value true. Later, if all Gadgets change to being not completed, then all actions will be called with the boolean value false.
allMustBeCompleted Typically, you will want all Gadgets on the list to be completed before the actions are invoked. Set this true when that's what you want. Setting it to false will result in the actions being invoked whenever the Gadgets are all not completed, and one of them becomes completed. Note that a second Gadget becoming completed does NOT invoke the list of actions again. It is only when all Gadgets are not completed, and one of them reports that it is completed, that the actions are invoked.
conditions Drag and drop the Gadgets that must be completed onto this array in the Unity Inspector.
numUncompleted This value tracks how many uncompleted conditions there are. It is public to enable debugging. Don't change it.
stayCompleted If you want the list to stay completed once its completion criteria are met, set this to true. Note that this does NOT force any of the Gadgets in the list to remain completed. Instead, it prevents the actions on the action list from being called again to signal that the criteria have gone from being met to unmet, even if one or more of the Gadgets changes from being completed to no longer being completed.
