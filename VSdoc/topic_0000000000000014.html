GadgetObserver Class

See Also

Abstract parent class for classes that present a human-friendly visual (or, possibly, audible) form of a Gadget. Whenever a Gadget's Interact method is called, whether it changes state or not, a GadgetObserver assigned to that Gadget will have its UpdateListener method called.

Note that the GadgetObserver presumes there is a Gadget on its GameObject, so it finds it and assigns the GadgetListener method to its OnUpdate event. Subclasses of this class will need to cast the Gadget to the particular subclass of Gadget they observe. They will also need to provide a concrete implementation of the UpdateListener method.

Start() Unity's Start method. Made protected and virtual so subclasses can override it but also call it.
UpdateListener(bool, int, int) Your subclass of GadgetObserver must implement this method to be notified whenever the Gadget you observe has its Interact method called. Note that the "isLatched" parameter is passed here as a reminder that, if a Gadget has its "stayCompleted" flag set true, and has ever been completed, you most likely don't want to update your observer in the normal way. More likely, you will want to present the human player with some indication that the this Gadget is no longer interactive.
